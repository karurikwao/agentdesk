# Code of Conduct

AgentDesk is meant to be a practical, welcoming project for people building and debugging AI agent systems.

## Expected Behavior

- Be respectful and specific when giving feedback.
- Assume good intent, but accept corrections quickly.
- Keep discussion focused on the work: bugs, design tradeoffs, safety, documentation, examples, and user impact.
- Help new contributors find a small next step when possible.
- Do not post secrets, private MCP configs, private paths, API keys, credentials, or confidential trace data.

## Unacceptable Behavior

- Harassment, threats, insults, or repeated personal attacks.
- Discriminatory language or exclusionary behavior.
- Publishing someone else's private information.
- Pressuring contributors to expose secrets, credentials, or private agent traces.
- Spam, low-effort promotion, or off-topic arguments that derail project work.

## Enforcement

Maintainers may edit or remove comments, close issues, block users, or restrict participation when behavior harms the project or its contributors.

If a report includes private information, do not post it in a public issue. Use GitHub private vulnerability reporting for security-sensitive conduct, secret exposure, or local-execution concerns, as described in [SECURITY.md](./SECURITY.md). For non-sensitive project moderation, open a GitHub issue with enough context for maintainers to act.

## Scope

This code applies to project spaces including GitHub issues, pull requests, discussions, release comments, and launch/community threads run by the maintainers.
